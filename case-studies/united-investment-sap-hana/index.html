The challenge

United Investment runs a mission-critical SAP HANA environment in Angola. Their ERP system supports finance, logistics, and operations across multiple business units. Moving it between Zadara clusters — even within the same data center region — required careful orchestration of 7 separate disk volumes without corrupting the HANA data layout or impacting production operations.

Previous attempts with traditional migration tools had failed: agent-based approaches couldn't safely handle the multi-disk HANA configuration, and manual disk-by-disk approaches were too slow and risky for a production environment.

Why Z-Move

  • Agentless: No software was deployed inside the SAP HANA VMs. The cloud API layer handled everything.
  • Multi-disk support: Z-Move's parallel migration engine handles multiple volumes per instance as a single atomic job.
  • Ephemeral workers: The migration infrastructure was auto-provisioned and auto-destroyed — no persistent compute left in either environment.

✓ Outcome: The SAP HANA environment was running in the target cluster within 253 minutes. No data corruption. No manual cleanup. The migration workers provisioned and destroyed themselves automatically — United Investment's team only needed to verify the application was healthy post-migration.

How it worked

Z-Move authenticated with both Zadara environments using API credentials. It discovered the SAP HANA VM and its 7 attached volumes, then provisioned ephemeral worker VMs in both source and target clusters. Each disk was streamed in parallel, with checksums verified on arrival. Once all 7 volumes were confirmed consistent, the target VM was rebuilt and powered on.

The entire process required zero interaction after the migration job was started. The team monitored progress on the Z-Move dashboard in real time.

What this means for similar workloads

SAP HANA is often cited as the hardest enterprise workload to migrate — it has strict disk layout requirements, large data volumes, and zero tolerance for inconsistency. Z-Move's approach proves that agentless migration at this complexity level is not only possible but reliable enough for production use.
